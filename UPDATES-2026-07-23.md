# 2026-07-23 – Remove the dead China events pipeline

## Context

The China monitor's "model catalysts vs US stock reactions" timeline card was removed
from `china-ai-monitor.html` on 2026-07-18 (`07679fa`) because its auto-generated half
depended on `XAI_API_KEY` (Grok Live Search), which was never configured – the card only
ever showed a frozen manual seed. The removal took out the card and its page JS, but left
the entire producer side of the pipeline in place:

- `.github/workflows/refresh.yml` still ran `update-china-events.py` every refresh
  (the step was key-gated, so it skipped – confirmed in run logs – but it was dead config),
  still uploaded `china-events.json` to R2, and still listed it in the data-commit loop;
- `worker.js` + `wrangler.jsonc` still routed `/china-events.json` worker-first to R2;
- `omen/china-events.json` (the 2026-07-16 manual Kimi K3 seed) was still served publicly;
- `omen/update-china-events.py` (the Grok generator with all its guardrails) sat unused.

Nothing on any page reads any of it. A data-source audit today flagged the half-state as
the worst option: dead weight that looks like a live feed.

## Change

Deleted every producer-side remnant:

- **`omen/update-china-events.py`** – deleted.
- **`omen/china-events.json`** – deleted (was the frozen manual seed).
- **`.github/workflows/refresh.yml`** – dropped the "Refresh China AI event timeline" step,
  the `put china-events.json` R2 upload, and `omen/china-events.json` from the commit list.
- **`worker.js`** – dropped the `/china-events.json` entry from `DATA_FILES`; comment
  no longer hardcodes a file count.
- **`wrangler.jsonc`** – dropped `/china-events.json` from `run_worker_first`; same
  comment fix.

Left alone: the `XAI_API_KEY` gate on the influencers step (that consumer is real – the
KOL board – and lights up if the key is ever added), and the historical
`UPDATES-2026-07-18.md` write-up of the original feature.

Not done here: the stale `china-events.json` object still sits in the `omen-data` R2
bucket. It is unreachable now (no worker route), so it is harmless; delete it manually
from the Cloudflare dashboard if you want the bucket clean.

## Restoring the timeline later

Everything is one revert pair away: `git revert 07679fa` brings the card + page JS back,
reverting this commit brings the generator + plumbing back, and the feature then needs
only the `XAI_API_KEY` repo secret. The guardrails (source-URL requirement, no invented
stock moves, 24h self-gate, union-dedupe, fail-safe on any error) are all in the reverted
script – see `UPDATES-2026-07-18.md` for the full design.
