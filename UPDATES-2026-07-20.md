# 2026-07-20 – China monitor: eight new data sources

Session goal: implement the full data-source expansion proposed in the 2026-07-19
research pass over the China AI Substitution Monitor. Every source below was probed
live before implementation; all shapes in `update-china-data.py` match real responses.

## What was added

All batch sources land in `china-data.json` (same override pattern as before); the
page renders them from `snap`, so nothing new is fetched client-side.

1. **Vercel AI Gateway lab share** (`vercel_gateway`) – the second router.
   `GET vercel.com/api/ai/leaderboard-export?dataset=labs&modality=text` (documented,
   CC BY 4.0, cached 24h server-side). Daily CN/US token-share series (last 90 days
   kept), top CN labs, own SPI. First reading: **SPI 1.01x vs OpenRouter's 1.97x** –
   exactly the de-bias the page needed. New card with splitbar + 60-day sparkline.
   The gauge's router family (w=30) now averages the OpenRouter and Vercel normalized
   shares when both are present; the SPI stat shows the Vercel cross-check as a subline.

2. **Ollama pull counts** (`ollama`) – local/self-hosted adoption, a segment no router
   or HF download counter sees. Parsed from the server-rendered `ollama.com/library`
   index (`ollama_models()`); CN/US classification by named model family
   (`OLLAMA_CN`/`OLLAMA_US` regexes; Mistral & friends deliberately unclassified).
   Lifetime totals, so a per-day rate is derived from a baseline in
   `china-history.json` once runs are >20h apart (same pattern as GitHub stars).
   New card + new gauge family `ollama` (w=10, 0→70% reference like the other shares).

3. **Kalshi prediction markets** (`kalshi`) – public trade-api v2, no key, but not
   CORS-open, so fetched by the updater. `KXBESTLLMCHINA` (Chinese model #1 before
   2027) plus the year-end `KXLLM1` event's Chinese brands and the single top-priced
   US brand for scale. Prices come from `last_price_dollars` (`last_price` is null on
   the public endpoint). Rows merge into the existing prediction-markets table with
   PM/KALSHI source tags; Kalshi rows render immediately from snap, Polymarket joins
   when its live fetch lands.

4. **LMArena official leaderboard dataset** – replaces the arena.ai HTML scrape as
   the *primary* leaderboard source (`lmarena_dataset()`, datasets-server rows API,
   config `text_style_control`, split `latest`, category `overall`; CC BY 4.0). The
   scrape stays as fallback; both funnel through the new pure `arena_summary()`.
   `lmarena.source` records which one won; `asof` is now the dataset's real publish
   date (2026-07-16 at time of writing) instead of the fetch date.

5. **Artificial Analysis Data API** – kills the manual snapshot when
   `ARTIFICIAL_ANALYSIS_API_KEY` is set (free tier, 1k req/day, attribution required
   – carried in the payload). Best CN vs best US by intelligence index
   (`aa_best()`), with the trends/apps-style carry-forward (`pick_aa()`); MANUAL
   remains the final fallback. **Mike: create a free key at
   artificialanalysis.ai/data-api and add it as a repo secret to go live.**

6. **HF fine-tune trees** (`hf_finetunes`) – ecosystem gravity. The exact tree size
   is only rendered on each base model's page ("N models" beside the Finetunes link
   – the list API paginates without totals, and models-json's numTotalItems is null
   now), so `parse_finetune_count()` scrapes it. Baskets: CN Qwen3-8B / DeepSeek-R1 /
   GLM-4.5-Air / Kimi-K2-Instruct vs US Llama-3.1-8B / gemma-3-12b-it / gpt-oss-20b /
   phi-4. First reading: CN 2,328 vs US 2,444.

7. **PyPI SDK downloads** (`sdk`) – pypistats.org weekly per-package
   (dashscope/zhipuai/zai-sdk vs openai/anthropic/google-genai), with 429 backoff.
   Honest caveat carried into the page note: Chinese APIs are OpenAI-compatible, so
   the CN side is a hard undercount – trend > level.

8. **Cloudflare Radar gen-AI service ranks** (`radar_ai`) – consumer traffic by
   1.1.1.1 DNS volume; actual visits, not search/app-chart proxies. Needs a free API
   token with Radar read scope in `CF_RADAR_TOKEN`; skipped cleanly without one and
   the page card stays hidden (`display:none`) until data exists. **Mike: mint a
   Radar-read token in the Cloudflare dashboard and add the secret to light this up**
   (the existing `CLOUDFLARE_API_TOKEN` is R2-scoped only – see UPDATES-2026-07-12).

## Gotchas for next time

- The Vercel export is cached 24h and its `share_percent` rows repeat per metric –
  filter `metric == "tokens"` or you sum requests/spend/imageCount into the share.
- Kalshi's public endpoint nulls `last_price`/`volume`; only `last_price_dollars`
  is populated.
- HF fine-tune counts: `models-json?other=base_model:finetune:X` still filters but
  `numTotalItems` is null (checked 2026-07-20); the model-page scrape is currently
  the only exact count.
- ollama.com pull counts render as `<span >117.3M</span><span>&nbsp;Pulls</span>` on
  the /library index only – individual model pages fetched via curl don't include
  the count markup.
